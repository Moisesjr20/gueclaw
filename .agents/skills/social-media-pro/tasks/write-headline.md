# Task: Write Headline (Criar 10 Headlines Testáveis)

**Input Necessário:**
- Produto/Serviço (o que está vendendo)
- Avatar (quem é o cliente ideal)
- Benefício principal (por que comprar)
- Stage of Awareness (Unaware/Problem/Solution/Product/Most Aware)

**Output Esperado:**
- 10 headlines A/B testáveis
- Variação de ângulos (diferentes abordagens)
- Score de qualidade >= 8.0

---

## 🎯 Objetivo

Criar **10 headlines de alto impacto** que param o scroll, criam curiosidade e conduzem à ação. Cada headline deve testar um **ângulo diferente** (benefit, curiosity, fear, proof, etc).

## 📋 Processo (Passo a Passo)

### **PASSO 1: ANALISAR CONTEXTO**

Antes de criar headlines, responda:

1. **Quem é o avatar?**
   - Idade, profissão, nível econômico
   - Dores principais
   - Desejos profundos

2. **Qual stage of awareness?**
   - Unaware? → Educar sobre problema
   - Problem Aware? → Apresentar solução
   - Solution Aware? → Diferenciar seu produto
   - Product Aware? → Oferta + social proof
   - Most Aware? → Urgência + escassez

3. **Qual o benefício #1?**
   - O que o cliente MAIS quer?
   - Como o produto entrega isso?

4. **Qual a objeção #1?**
   - O que impede a compra?
   - Como remover isso?

### **PASSO 2: CHAMAR SPECIALISTS**

**Para audiences Stage 1-3 (frio/morno):**
- Gary Halbert (WIIFM, benefit direto)
- Eugene Schwartz (awareness stages, mass desire)

**Para audiences Stage 4-5 (quente):**
- Dan Kennedy (urgência, scarcity)
- Joe Sugarman (psychological triggers)

**Para produtos técnicos/complexos:**
- Clayton Makepeace (mechanism, proof)
- David Ogilvy (autoridade, long-form)

### **PASSO 3: GERAR 10 ÂNGULOS DIFERENTES**

Use esta diversificação de ângulos:

1. **Benefit Direto** (Gary Halbert style)
   - "Como [Benefício] em [Prazo]"

2. **Curiosity Gap** (Eugene Schwartz style)
   - "O Segredo de [Grupo Desejado] Para [Desejo]"

3. **Fear/Pain** (Clayton Makepeace style)
   - "Por Que [Consequência Negativa] Se [Não Agir]"

4. **Proof Social** (Joe Sugarman style)
   - "[Número] Pessoas [Resultado] Usando [Produto]"

5. **Mechanism** (Todd Brown style)
   - "O Sistema [Nome] Que [Resultado Específico]"

6. **Contrarian** (Ben Settle style)
   - "Por Que [Crença Comum] Está ERRADA (E O Que Fazer)"

7. **Identity** (Dan Koe style)
   - "Para [Identidade Desejada] Que Querem [Específico]"

8. **Urgency** (Dan Kennedy style)
   - "[Benefício] Antes Que [Consequência] / [Prazo]"

9. **Newness** (David Ogilvy style)
   - "A Nova Forma de [Solução] Que [Diferencial]"

10. **Question** (Gary Bencivenga style)
    - "[Pergunta Provocativa Que Prospect Quer Resposta]"

### **PASSO 4: REFINAR CADA HEADLINE**

Para cada headline, aplique:

**Checklist de Qualidade:**
- [ ] **Clareza** - É óbvio o que você oferece?
- [ ] **Especificidade** - Tem números/detalhes concretos?
- [ ] **Benefício** - WIIFM ("What's In It For Me?")?
- [ ] **Curiosidade** - Cria vontade de saber mais?
- [ ] **Credibilidade** - Soa realista (não exagerado)?
- [ ] **Urgência implícita** - Por que agir AGORA?

**Teste Halbert:** Minha avó de 80 anos entenderia? Se não, simplifique.

**Teste Schwartz:** Alinhado ao stage of awareness? Se não, ajuste.

### **PASSO 5: VALIDAR SCORE >= 8.0**

Cada headline recebe score baseado em:

- **Impact** (9+/10) - Para o scroll em 3 segundos?
- **Clarity** (9+/10) - Óbvio o que oferece?
- **Benefit** (8+/10) - WIIFM explícito?
- **Curiosity** (7+/10) - Quer saber mais?
- **Credibility** (8+/10) - Acredita que é possível?

**Score mínimo aprovação: 8.0/10**

Se < 8.0 → Refinar e re-avaliar
